<objective>
Close the blocker gap from 21-UAT.md test 8: `RecruitEvents` (and sibling recruits-side event handler classes) are never registered on the Forge event bus after the Phase-21 consolidation, so their `onServerStarting` hooks never fire, which leaves static manager fields (`recruitsPlayerUnitManager`, `recruitsClaimManager`, `recruitsFactionManager`, etc.) null. First interaction that touches any of those — currently right-clicking a recruit to hire — trips an NPE on the server.

Fix: in `BannerModMain.setup(FMLCommonSetupEvent)`, register every recruits-side `@SubscribeEvent` handler class on `MinecraftForge.EVENT_BUS` exactly once, alongside the existing Workers registrations. This closes the reported NPE and the identical latent NPE class for the sibling handlers in a single atomic patch.

Purpose: resume UAT test 8 (network packet round-trip — recruit command + work area update) and unblock phase-21 sign-off.
Output: single modified file (`BannerModMain.java`) + user-verified right-click-to-hire flow.
</objective>

Forge registration contract: `MinecraftForge.EVENT_BUS.register(Object target)` reflects over `target`'s (or its class's) `@SubscribeEvent`-annotated methods. Passing `new X()` works for classes whose handlers are instance methods, static methods, or a mix. Each class must be registered exactly once per JVM.

All target classes live in package `com.talhanation.bannermod.events`. None take constructor arguments (default no-arg constructors; already used by existing gametests: `new FactionEvents()`, `new RecruitEvents()`, `new ClaimEvents()`).

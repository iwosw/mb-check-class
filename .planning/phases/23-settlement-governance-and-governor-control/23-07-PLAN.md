<objective>
Guard all `RecruitsClientConfig.RecruitsLookLikeVillagers.get()` calls in entity sound/cosmetic methods against the config-not-loaded state that the gametest harness exposes during `LivingEntity.baseTick`.

Root cause: `ForgeConfigSpec.ConfigValue.get()` throws `IllegalStateException("Cannot get config value before config is loaded")` when called before Forge fires `ModConfigEvent`. The gametest harness ticks entities immediately after spawn, before config loads. Six call sites across two files hit this path.

Fix: wrap each `.get()` in a try-catch or isLoaded guard, falling back to the config's declared default (`true` for `RecruitsLookLikeVillagers`). This is a zero-risk defensive fix — normal gameplay always has config loaded; only test harnesses hit the early-tick path.
</objective>

<tasks>

<task id="1" name="Guard config reads in AbstractRecruitEntity and AbstractWorkerEntity">

**What:** Add a private static helper `isVillagerAppearance()` to each entity class (or a shared utility) that wraps `RecruitsClientConfig.RecruitsLookLikeVillagers.get()` in a try-catch returning `true` (the config default) on `IllegalStateException`. Replace all 6 bare `.get()` calls with the guarded helper.

**Call sites to fix:**

| File | Line | Method |
|------|------|--------|
| `AbstractRecruitEntity.java` | 1089 | `getHurtSound` |
| `AbstractRecruitEntity.java` | 1093 | `getDeathSound` |
| `AbstractRecruitEntity.java` | 1989 | `makeLevelUpSound` |
| `AbstractRecruitEntity.java` | 1994 | `makeHireSound` |
| `AbstractWorkerEntity.java` | 514 | `getHurtSound` |
| `AbstractWorkerEntity.java` | 523 | `getDeathSound` |

**Implementation approach:**
Add a package-private static helper (e.g., in `AbstractRecruitEntity` since both files are in sibling packages):
```java
static boolean recruitsLookLikeVillagers() {
    try {
        return RecruitsClientConfig.RecruitsLookLikeVillagers.get();
    } catch (IllegalStateException e) {
        return true; // config default
    }
}
```
Then replace each bare `RecruitsClientConfig.RecruitsLookLikeVillagers.get()` with the helper call. `AbstractWorkerEntity` can either duplicate the helper or call the one in `AbstractRecruitEntity` depending on package visibility.
